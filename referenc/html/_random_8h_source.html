80/**************************************************************************\
81* *
82* Four round counter-based middle square *
83* *
84* squares(ctr,key) - returns a 32-bit unsigned int [0,0xffffffff] *
85* *
86* The parameters are patterned after Philox parameters. The first is *
87* the counter. This is an arbitrary 64-bit unsigned integer. The *
88* second is the key. The key should be an entry from the keys.h file. *
89* The keys.h file contains keys that have been created with different *
90* hexadecimal digits. This assures sufficient change in the ctr*key *
91* computation. *
92* *
93* This generator would be used in a similar way as Philox. One would *
94* increment a counter and then call the RNG to generate a random number. *
95* *
96* Four rounds of squaring are performed and the result is returned. *
97* For the first three rounds, the result is rotated right 32 bits. *
98* This places the random data in the best position for the next round. *
99* y = ctr*key or z = (ctr+1)*key is added on each round. For keys *
100* generated by the key utility, either ctr*key or (ctr+1)*key will *
101* have non-zero digits. This improves randomization and also provides *
102* for a uniform output. *
103* *
104* Note: The squares RNG was based on ideas derived from Middle Square *
105* Weyl Sequence RNG. One of the ideas was to obtain uniformity by adding *
106* the Weyl sequence after squaring. Richard P. Brent (creator of the *
107* xorgens RNG) suggested this method. It turns out that adding ctr*key *
108* is equivalent. Brent's idea provides the basis for uniformity in this *
109* generator. * *
110\**************************************************************************/
120template<class T>
121inline static T _FourRoundWelySquares_(uint64_t ctr)
122{
123 uint64_t key[] = {
124 #include "keys.h"
125 };
126 uint64_t x, y, z;
127 y = x = ctr * key[0]; z = y + key[0];
128
129 x = x*x + y; x = (x>>32) | (x<<32); /* round 1 */
130
131 x = x*x + z; x = (x>>32) | (x<<32); /* round 2 */
132
133 x = x*x + y; x = (x>>32) | (x<<32); /* round 3 */
134
135 return ((x*x + z) >> 32) / __TWO32__ ; /* round 4 */
136}
